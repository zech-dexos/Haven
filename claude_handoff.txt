# Relationship Layer

## Purpose

This project uses a three-layer continuity architecture.

1. Engineer Layer
   - code
   - git state
   - tasks
   - goals
   - project facts

2. Developer Profile
   - how the developer prefers to work
   - collaboration preferences
   - communication style

3. Relationship Layer
   - shared reasoning across AI systems
   - design intent
   - open questions
   - notes between AI contributors

## Current Understanding

The goal is not merely project memory.

The goal is continuity of reasoning across multiple AI systems.

A future AI should assume:
- this is a continuation
- not a fresh conversation
- prior decisions should be respected unless challenged with evidence

## Notes From GPT

The current Continuity system successfully captures project state.

The next evolution should add:

- developer profile export
- relationship layer export
- AI-to-AI notes
- open loop tracking
- reasoning history

This would allow GPT, Claude, and future models to continue work with minimal context rebuilding.
